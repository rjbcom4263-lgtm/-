# Saju Engine Roadmap

이 프로젝트는 `manseryeok`를 기준 엔진으로 시작하고, `ssaju`는 비교/확장 엔진으로 천천히 도입한다.

## Stage 1. 기준 만세력 엔진

목표: 생년월일시 입력에서 흔들리지 않는 기본 명식 데이터를 만든다.

- 기준 엔진: `manseryeok`
- 산출 항목: 사주팔자, 십성, 공망, 대운, 오행 분포
- 검증 방식: 대표 사주 케이스를 `vitest`로 고정
- 현재 상태: 진행 중

완료 조건:

- `calculateSajuChart()`가 화면과 분리된 서비스용 결과를 반환한다.
- 대표 케이스의 연주, 월주, 일주, 시주가 테스트로 고정된다.
- 십성, 공망, 대운 시작 나이, 오행 분포가 테스트로 고정된다.

## Stage 2. 만세력 표 확장

목표: 포스텔러 1페이지 수준의 명식표에 필요한 보조 항목을 추가한다.

- 지장간
- 12운성
- 12신살
- 천간/지지별 길성
- 합충형파해 기본 감지

완료 조건:

- 네 기둥마다 천간, 지지, 십성, 지장간, 12운성, 12신살을 구조화한다.
- 항목별 계산 규칙을 독립 함수로 분리한다.
- 표준 케이스와 경계 케이스 테스트를 추가한다.

## Stage 3. `ssaju` 비교 도입

목표: `ssaju`를 정답으로 무조건 채택하지 않고, 보조 검증 엔진으로 쓴다.

- `manseryeok` 결과와 `ssaju` 결과를 같은 입력으로 비교한다.
- 일치 항목은 자동 테스트로 고정한다.
- 불일치 항목은 관법 차이인지 버그인지 기록한다.
- 쓸 만한 항목은 우리 `SajuChart` 구조로 흡수한다.

완료 조건:

- 비교 리포트 스크립트가 있다.
- 차이가 나는 항목은 `ENGINE_DECISIONS.md`에 기준을 남긴다.

## Stage 4. 서비스형 해석 레이어

목표: 계산 결과를 사용자에게 읽히는 해석으로 바꾼다.

- 오행/십성 분석
- 신강/신약 지수
- 용신 후보
- 대운/세운/월운 흐름
- 성격, 직업, 재물, 관계 해석

완료 조건:

- 계산 데이터와 해석 문장이 분리되어 있다.
- 해석은 버전 관리 가능한 규칙/콘텐츠로 작성된다.
- 면책 문구와 라이선스 고지가 포함된다.
