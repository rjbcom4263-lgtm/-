# 운세 문장 톤 분리 및 10개 샘플 테스트 보고서

작성일: 2026-07-03

## 1. 작업 목적

오늘/내일/주간/월간 운세 문장이 데이터 나열처럼 보이거나, 조사 연결이 어색해지는 문제를 줄이기 위해 운세 문장 톤을 3가지로 분리했다.

문장팩을 새로 늘리는 방식이 아니라, 같은 계산 결과를 어떤 사용자에게 보여줄지에 따라 다르게 표현하는 구조로 만들었다.

## 2. 적용한 톤

| 톤 | 목적 | 문장 성격 |
| --- | --- | --- |
| 부드러운 톤 | 초보자, 일반 사용자 | 부담 없이 읽히는 짧고 쉬운 조언 |
| 현실 조언 톤 | 기본 화면용 | 점수, 상위 퍼센트, 기회, 주의, 일정 배치를 함께 설명 |
| 전문가 톤 | 검증/상세 보기용 | 십성, 오행, 우세 카테고리, 활용일, 주의일 중심 |

현재 화면 기본값은 기존처럼 현실 조언 톤이다. 부드러운 톤과 전문가 톤은 데이터에 같이 생성되며, 이후 UI에서 선택형으로 열 수 있다.

## 3. 수정 내용

- `DailyFortuneReading`에 `toneVariants` 추가
- `PeriodFortuneReading`에 `toneVariants` 추가
- 오늘 운세 톤 3종 생성
- 주간/월간 운세 톤 3종 생성
- `정재, 편관 십성과 목, 화 오행` 같은 나열형 문장을 의미 중심으로 변경
- 날짜 표기를 `2026-07-01 병자`에서 `7월 1일(병자)` 형식으로 단축
- 조사 오류 방지
  - `이번 주은` 방지
  - `월간 운세은` 방지
  - `정리을` 방지

## 6. 발견 및 수정한 문제

초기 10개 샘플 테스트 중 전문가 톤에서 다음 문제가 발견됐다.

```text
월간 운세은 평균 64점...
```

원인:

`title + 은` 방식으로 조사를 고정해 붙였기 때문이다.

수정:

```text
월간 운세는 평균 64점...
```

전문가 톤에서는 `title + 는`으로 통일해 자연스럽게 처리했다.

## 7. 샘플 문장 비교

### 부드러운 톤

